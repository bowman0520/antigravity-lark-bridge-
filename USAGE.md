# Antigravity 飞书桥接 - 通用异步长任务工具说明

本系统支持长任务（如视频剪辑、大型构建、复杂推理等）的异步化处理，以防止执行阻塞导致看门狗超时强杀。

## 什么是 `run_async.js`？
它是一个长任务的**后台包装脚本**。
任何原本要通过 `run_command` 或其他同步调用的重任务，都可以交由 `run_async.js` 来执行。它会把你的任务放进隐形的后台进程，并在执行完成后，利用飞书桥接层独有的 `taskWatcher`，自动唤醒你并报告执行结果。

## 什么时候使用它？
只要你预估你的命令行任务会执行超过 **15秒**（例如：ffmpeg 视频渲染、调用远程大模型处理大文件、编译大型项目），就**必须**使用本工具。

## 如何使用？

使用 Node.js 运行项目根目录下的 `run_async.js` 脚本：

```bash
node run_async.js "<scope>" <实际要执行的命令和参数...>
```

### 参数说明：
- `scope`: **必填**。代表当前会话的作用域。请从你收到用户指令的 `<bridge_context>` 块中找到 `scope: ` 字段的值，然后**原样**填入此处（例如 `"p2p:ou_123456"` 或 `"chat:oc_abcdef"`）。注意，必须用双引号包起来，防止特殊字符被 Shell 截断。
- `<实际要执行的命令>`: 剩下你需要跑的命令和参数。

### 示例用法：

如果你原来的命令是：
```bash
ffmpeg -i input.mp4 -vcodec libx264 output.mp4
```

你需要将其改为：
```bash
node run_async.js "p2p:ou_123456" ffmpeg -i input.mp4 -vcodec libx264 output.mp4
```

## 执行机制：
1. 当你执行上述命令后，它会**立刻成功退出**（通常只耗时几十毫秒），你的终端会显示类似：`[成功] 后台任务已启动 (Scope: p2p:xxxx)。主进程已释放...`。
2. 此时你应该**结束当前回合**，并通过向用户发消息告知：“任务已开始在后台运行，完成后我会通知您”。
3. 当后台的 `ffmpeg`（或其它任务）真正运行结束后，桥接系统会主动在控制台向你**注入一条系统通知**（带上 stdout 和 stderr 结果）。
4. 收到通知被唤醒后，你需要审查执行日志，然后把最终情况反馈给飞书上的用户。
